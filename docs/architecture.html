Architecture

Python Snapline is a uv workspace monorepo mirroring the Node.js package split. Consumers install snapline-core; config keys stay camelCase for cross-language parity.

Snapline Hub (optional)

Snapline Hub is a separate, optional reporting UI for TestRunReport data. Not required for Snapline — use when you want a centralized dashboard. Push via push_test_report_to_hub() or SNAPLINE_HUB_URL. The Node.js edition has equivalent pushTestReportToHub().

Published package dependencies

snapline-core depends only on httpx. It exports DbConnectionLike for SQL — you bring your own Postgres/MySQL driver. SQLite and stub DB helpers live in snapline.demo_shared (demo only).

Differences from Node

  • No build step — run demos directly with uv run demo
  • Async test_suite — use asyncio.run() or await in async test runners
  • httpx — HTTP client with 30s default timeout (Node uses fetch + AbortSignal)
  • Same scenario IDs — all 21 demo ids match the Node repo
